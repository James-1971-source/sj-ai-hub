# S&J희망나눔 AI 도구 허브 — 배포 가이드

이 문서는 `index.html` 한 파일로 완성된 AI 도구 허브를 실제 운영 사이트로 배포하고,
로고 표시와 SNS 실제 로그인을 활성화하는 전체 절차를 안내합니다.

---

## 1. 파일 구성

```
배포 폴더/
├── index.html   ← 사이트 전체 (이 파일 하나로 동작)
└── logo.png     ← 법인 로고 이미지 (직접 추가)
```

- `logo.png`를 `index.html`과 **같은 폴더**에 넣으면 로고가 자동으로 표시됩니다.
- 파일이 없으면 S&J 그린 + 골드 SVG 엠블럼이 대신 표시됩니다 (오류 없음).
- 권장 로고 사양: 정사각형 PNG, 200×200px 내외, 100KB 이하.
- 기존 Base64 방식의 용량 제한 문제는 외부 파일 참조 방식으로 완전히 해결되었습니다.

## 2. 무료 호스팅 (백엔드 서버 불필요)

### 방법 A — GitHub Pages (권장)
1. github.com 에서 새 저장소 생성 (예: `sj-ai-hub`)
2. `index.html`과 `logo.png` 업로드
3. 저장소 Settings → Pages → Branch를 `main`으로 설정 → Save
4. 1~2분 후 `https://계정명.github.io/sj-ai-hub/` 주소로 접속 가능

### 방법 B — Netlify
1. netlify.com 가입 → "Add new site" → "Deploy manually"
2. 두 파일이 든 폴더를 드래그앤드롭
3. 즉시 `https://임의이름.netlify.app` 주소 발급

> 법인 도메인(예: `ai.sj-hs.or.kr`)을 연결하려면 두 서비스 모두
> 무료로 커스텀 도메인 설정을 지원합니다 (DNS에 CNAME 레코드 추가).

## 3. SNS 실제 로그인 활성화

`index.html` 상단의 `CONFIG` 객체에 키 4개만 입력하면 됩니다.

```javascript
const CONFIG = {
  GOOGLE_CLIENT_ID: "여기에 입력",
  KAKAO_JS_KEY:     "여기에 입력",
  NAVER_CLIENT_ID:  "여기에 입력",
  NAVER_CALLBACK:   "https://배포주소/index.html"
};
```

키를 입력하지 않으면 해당 버튼은 데모 안내만 표시합니다 (오류 없음).
3개 키를 모두 입력하면 모달 하단의 데모 안내문이 자동으로 사라집니다.

### 3-1. Google
1. https://console.cloud.google.com → 새 프로젝트 생성
2. "API 및 서비스" → "OAuth 동의 화면" → 외부 → 앱 이름·이메일 입력
3. "사용자 인증 정보" → "OAuth 클라이언트 ID 만들기" → 유형: **웹 애플리케이션**
4. **승인된 자바스크립트 원본**에 배포 주소 등록
   (예: `https://계정명.github.io`)
5. 발급된 **클라이언트 ID**를 `GOOGLE_CLIENT_ID`에 입력

### 3-2. Kakao
1. https://developers.kakao.com → "애플리케이션 추가"
2. 앱 키 중 **JavaScript 키**를 `KAKAO_JS_KEY`에 입력
3. "플랫폼" → Web → 사이트 도메인에 배포 주소 등록
4. "카카오 로그인" 활성화 → **Redirect URI**에 배포된 페이지 전체 주소 등록
   (예: `https://계정명.github.io/sj-ai-hub/index.html`)
5. "동의 항목"에서 닉네임(필수), 이메일(선택) 설정

### 3-3. Naver
1. https://developers.naver.com → "애플리케이션 등록"
2. 사용 API: **네이버 로그인** 선택, 제공 정보: 이름·이메일
3. 서비스 환경: PC 웹 → 서비스 URL과 **Callback URL** 등록
   (Callback URL = 배포된 페이지 주소, `NAVER_CALLBACK`에도 동일하게 입력)
4. 발급된 **Client ID**를 `NAVER_CLIENT_ID`에 입력

## 4. 현재 구현 수준과 다음 단계

| 기능 | 현재 상태 | 비고 |
|---|---|---|
| 카드 디렉토리, 검색, 필터 | ✅ 완성 | 24개 도구 등록 |
| 다크/라이트 모드, 한/영 전환 | ✅ 완성 | 설정 자동 저장 |
| 로고 표시 | ✅ 해결 | logo.png 외부 참조 |
| SNS 로그인 (Google/Kakao/Naver) | ✅ 실제 연동 코드 탑재 | 키 입력 시 즉시 작동 |
| 이메일 회원가입/로그인 | ⚠️ 브라우저 저장 방식 | 기기별 저장, 데모 수준 |

이메일 회원 정보는 사용자 브라우저(localStorage)에만 저장되므로
**실제 회원 DB가 필요해지면** Firebase Authentication 또는 Supabase(무료 플랜)
연동을 다음 단계로 권장합니다. 요청하시면 해당 연동 버전을 제작해 드립니다.

## 5. 유의 사항

- claude.ai 미리보기 환경에서는 보안 정책상 localStorage와 SNS 팝업이
  제한되므로, **로그인 기능의 완전한 테스트는 배포 후 실제 주소에서** 진행하세요.
- 카카오 로그인은 등록된 Redirect URI와 실제 주소가 한 글자라도 다르면
  KOE006 오류가 발생합니다. http/https, 끝의 슬래시까지 정확히 일치시켜 주세요.
- 도구 추가/삭제는 `index.html`의 `TOOLS` 배열에 한 줄씩 추가·삭제하면 됩니다.
